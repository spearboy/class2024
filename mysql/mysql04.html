Primary key(기본 키)는 관계형 데이터베이스에서 테이블 내의 각 레코드를 고유하게 식별하기 위해 사용되는 필드(열)나 필드 집합(여러 개의 열)입니다. Primary key는 테이블의 레코드를 식별하고 구별하는 데 사용되며, 각 레코드는 오직 하나의 Primary key 값을 가질 수 있습니다.

Primary key의 주요 특징은 다음과 같습니다:
1. 고유성 (Uniqueness): Primary key의 값은 테이블 내에서 모든 레코드에 대해 고유해야 합니다.
    즉, 중복된 값이 없어야 합니다.
2. 최소성 (Minimality): Primary key는 가능한 한 최소한의 필드로 구성되어야 합니다.
    필요 이상의 필드가 포함되어서는 안 됩니다.
3. NULL 값 허용 불가능 (Non-nullability): Primary key는 NULL 값을 가질 수 없습니다.
    각 레코드는 Primary key 값을 가져야 합니다.
4.변하지 않음 (Immutability): Primary key 값은 변경되어서는 안 됩니다.
    특히, 이미 존재하는 레코드의 Primary key 값은 변경되면 안 됩니다.

일반적으로 Primary key는 테이블 내의 한 열에 의해 정의됩니다. 그러나 경우에 따라 여러 열의 조합으로 구성될 수도 있습니다. Primary key는 데이터베이스의 인덱스 생성에 사용되며, 효율적인 데이터 검색 및 정렬을 지원합니다.

create table members(
->     memberID int(10) unsigned auto_increment,
->     youEmail varchar(40) NOT NULL,
->     youName varchar(10) NOT NULL,
->     youPass varchar(20) NOT NULL,
->     youBirth int(10) NOT NULL,
->     youAge int(5) NOT NULL,
->     regTime int(30) NOT NULL,
->     PRIMARY KEY(memberID)
-> ) charset=utf8;
